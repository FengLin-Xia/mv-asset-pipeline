## 3. 单仓库交接卡片

### 仓库 1：`mv_case_pipeline`

#### 3.1 项目用途

该仓库用于将 MV 视频自动拆解为结构化资产，输出镜头切分、关键帧（含颜色调性）、音乐段落结构、视觉节奏关联、人声标注、视觉 Caption 和四层 JSON 资产表，供后续 AI 辅助 MV 创作使用。

#### 3.7 启动方式

```bash
# 1. 进入项目目录
cd c:\Users\vip\Desktop\BIG_PROGRAM\mv_case_pipeline

# 2. 安装主环境依赖（Windows 终端）
conda env create -f environment.yml
conda activate mvcase
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu124

# 3. （可选）安装 demucs 用于人声分离
pip install demucs

# 4. 配置环境变量（Step 5 Caption 需要）
cp .env.example .env
# 编辑 .env，填入 VLM_API_KEY

# 5. （Step 4 需要）在 WSL Ubuntu 中初始化 SongFormer
cd /mnt/c/Users/<用户名>/Desktop/BIG_PROGRAM/mv_case_pipeline/step4_songformer
chmod +x setup_wsl_env.sh
./setup_wsl_env.sh
# 然后手动下载 ckpts 到 ~/projects/SongFormer/src/SongFormer/ckpts/

# 6. 将 MV 视频放入 data/raw/，运行全流程
python src/main.py --input data/raw/MV_001.mp4 --mv-id MV_001

# 快速验证（跳过 GPU 和 API）
python src/main.py --input data/raw/MV_001.mp4 --mv-id MV_001 --skip-demix --skip-music --skip-caption --debug
```

#### 3.12 已知问题 / 坑点

| 问题 | 影响 | 建议处理 |
|---|---|---|
| ckpts 放错目录 | Step 4 报「缺少必要的 ckpts 文件」 | 必须放在 `src/SongFormer/ckpts/`，不是仓库根 `ckpts/` |
| WSL GPU 不可用 | Step 4 无法推理 | Windows 侧 `wsl nvidia-smi` 验证，驱动需 ≥ 470.76 |
| Step 4 warning 很多 | 易误判为失败 | 只要生成了非空 `music_structure_raw.json` 即可忽略 |
| demucs 未安装 | Step 1.5 失败，`has_vocals` 为 null | `pip install demucs` 或 `--skip-demix` |
| 无 API Key | Step 5 失败 | 配置 `.env` 或 `--skip-caption` |
| `data/processed/` 不在 Git | 克隆仓库后无样例产出 | 单独交接 MV_001–MV_006 数据 |
| 极短 clip 关键帧提取失败 | ffmpeg 返回 EINVAL，时间戳超出最后一帧 | 已修复：重试上限改为 `duration - 1/fps` |

## 7. 风险与注意事项汇总

### 7.1 版本风险

- 当前唯一主线版本为 Git `master` 分支，远程：https://github.com/FengLin-Xia/mv-asset-pipeline.git
- 本地 `data/processed/` 产出不在 Git 中，克隆仓库不等于有样例数据
- `TODO.md` 描述的部分功能已在代码中实现，勿按 TODO 重复开发

### 7.2 环境风险

- **强依赖 Windows + WSL2 双环境**：Step 4 无法在纯 Windows 主环境中运行
- **GPU 依赖**：Step 4 SongFormer 需要 WSL 内 NVIDIA GPU；CPU 模式不支持 Step 4
- **Python 双 conda 环境**：Windows 侧 `mvcase` + WSL 侧 `songformer`，不可混用
- **PyTorch 需按 CUDA 版本手动安装**，未锁定在 `environment.yml` 中
- 目前仅在 `vip` 用户本机（Windows + WSL Ubuntu 22.04）完整跑通过

### 7.3 权限风险

- Step 5 需要 VLM API Key（`.env` 文件，不入 Git）
- SongFormer ckpts 从 HuggingFace / ModelScope 下载，可能需要网络访问
- MV 原始视频和处理产出可能涉及版权，**不宜随意外传或公开上传**
- GitHub 仓库为公开/私有状态需确认（当前 remote 已配置）

### 7.4 业务理解风险

- 四层 JSON 资产表字段含义和下游消费方式需与业务侧确认
- `configs/label_taxonomy.yaml` 中的标签枚举（音乐段落、镜头功能等）可能被业务调整
- `shot_review.csv` 中 `needs_review=True` 表示需人工复核，自动化 Caption 不可直接作为最终资产
- 音乐段落 label（intro/chorus 等）到中文（开场/副歌等）的映射逻辑在 Step 6 中，变更需同步下游

## 9. 给接手人的说明

当前唯一主线是 **mv_case_pipeline**——一个将 MV 视频拆解为结构化 AI 创作资产的 Python 工具链，最终输出案例级总结（`case_summary.json` + `case_summary.md`），为后续案例库检索和 AI 辅助创作提供结构化输入。

请先阅读 `README.md`，用以下命令快速批量验证全流程（无需 GPU 或 API Key）：

```bash
python src/batch_run.py --skip-demix --skip-music --skip-caption --no-llm
```

`data/raw/` 下的视频会自动处理为 MV_001–MV_006，并生成规则版 `case_summary`。再逐步配置 WSL SongFormer（Step 4）、VLM API（Step 5/7）完成完整分析链。本地产出不在 Git 中，需单独获取。
